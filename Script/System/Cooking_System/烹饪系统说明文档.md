将目前的制作食物系统进行重构，变为一个新的整体的烹饪系统，包括以下几个方面：
1. 将所有与食物制作相关的文件和代码都移动到该目录下： 'Script\System\Cooking_System' 以便于管理和维护。包括且不限于以下已知相关内容：'Script\UI\Panel\make_food_panel.py'、'Script\Design\cooking.py'、'Script\UI\Panel\food_bag_panel.py'。
2. 当前的食物中还有食材、食物重量等已经废弃不用的内容，整理出所有已弃用的内容，并给出清理和优化的方案。包括函数的优化、变量的优化、数据结构的优化等。
3. 制作食物的时候，目前只能一次制作一个食物，增加批量制作功能，允许玩家一次性制作多个相同的食物。制作多个食物的时候会有时间的减少，每多制作一个食物，所需要多花费的时间与菜谱的难度等级有关。比如1级难度的菜谱，每多制作一个食物，所需时间增加1分钟；2级难度的菜谱，每多制作一个食物，所需时间增加2分钟；3级难度的菜谱，每多制作一个食物，所需时间增加3分钟，以此类推。
4. 根据菜谱等级，制作更高级的食物时，会获得更多的料理经验和更高的习得值。当前制作食物的结算是在变成制作食物行为之后，调用'data\csv\Behavior_Effect.csv'中'134,make_food'里的结算id进行结算。该结算里经验的获得(即'CVE_A1_E|83_G_1'等)是固定的，习得的获得（1751）也是固定的。需要改为读取当前的菜谱等级来动态获得，具体方式是仿照其他的指令专属动态结算（即'Script\Core\constant_effect.py'中的'指令_专用结算'），创建一个单独的结算来代替上述的经验和习得结算。在该单独结算中读取玩家当前行为中的菜谱等级来进行结算。
5. 之前的旧代码中可能有以下有问题的食物创建逻辑，你需要判断该问题是否依然存在，如果不存在则无需处理，如果存在则需要处理：角色制作食物时的逻辑是，先创建所有角色可以制作的食物对象，然后角色在选中要制作的食物菜谱时，直接获取对应的食物对象进行制作。现在需要改为，角色在选中要制作的食物菜谱时，才创建对应的食物对象进行制作。也就是说，制作食物的对象创建逻辑需要从'初始化'阶段移到'制作食物'阶段。使得在制作食物时，只有当前选中的菜谱对应的食物对象才会被创建，其他未选中的食物对象不会被创建，从而减少不必要的内存占用和性能消耗。
6. 做非特殊调味的饭的时候可以选择标准模式和精细模式。标准模式下的品质有上限，品质的基础计算方式和当前一样，但是上限只能到普通。精细模式下的品质没有上限，品质的基础计算方式和当前一样并最多只到普通，但在回答问题时如果答对了可以提升品质。
- 精细模式下需要回答烹饪时的问题，按照烹饪的阶段，依次需要回答以下几个方面的每个方面一个问题：备料时、烹饪时、调味时、装盘时。每个阶段的问题都是随机的，从该食物的问题库中随机选择该阶段的问题。每个问题有多个选项，选项也为随机排列使每次遇到该问题的时候选项顺序不相同。问题的选项为单选，其中都有一个正确答案，其他为错误答案。选择正确答案会增加食物品质，选择错误答案不会减少食物品质。每答对一个问题，食物品质增加1，最多不超过上限（8）。
- 每个食物都有一个单独的烹饪问题库，问题库中包含了每个阶段的问题和答案。每个食物的烹饪问题库都是独立的，互不影响。每个食物的每个阶段的问题库中至少包含10个问题，每个问题有4个选项，其中一个为正确答案，其他为错误答案。错误答案中有两个错误答案需要非常合理，与正确答案有一定的迷惑性，使得玩家在选择时需要仔细思考，最后的一个错误答案可以是一个明显错误的选项，用于测试玩家的基础知识。
- 问题库为csv数据文件，参考'data\csv\Medical_Complication.csv'的格式，存放在'data\csv\cook_question'目录下，文件名为'<食物ID>_cook_question.csv'。第一列为行cid，第二列为食物id，第三列为阶段（备料、烹饪、调味、装盘），第四列为问题内容，第五列为正确答案，第六列为错误答案1，第七列为错误答案2，第八列为错误答案3。你需要在任务的最后再单独调用多轮子代理来生成该csv文件，以免该较大的数据库文件占用过多的上下文空间。每个子代理都会打开并修改该csv文件，生成一个目标食物的烹饪问题库，该子代理生成完毕之后，再顺序调用下一个子代理进行下一个食物的生成，不能同时调用多个子代理同时进行生成。
- 需要生成烹饪问题库的食物见菜谱文件（data\csv\Recipe.csv），该文件中以下条件的食物是不可烹饪的，所以也不用生成烹饪问题库：
  - 烹饪难度（difficulty）大于等于100的食物
  - 菜谱类型为“5预制食物”的食物
  - 菜谱类型为“8加料咖啡”的食物
  - 菜谱类型为“9其他”的食物
- 在食物制作面板（Script\System\Cooking_System\make_food_panel.py）中，在调味选择的下方，食物选择的上方，增加烹饪模式的选择UI。同样为三行，第一行为标题说明，第二行为标准模式的选择按钮以及标准模式的说明，第三行为精细模式的选择按钮以及精细模式的说明。默认为标准模式。在调味不是特殊调味的情况下，精细模式是可选的，否则精细模式不可选，且显示为灰色不可点击状态。
- 在选择了精细模式后，制作食物的过程中会显示出烹饪问题的回答界面。每次依次显示一个阶段的问题，玩家需要选择一个答案后才能进入下一个阶段的问题。每个阶段的问题回答完毕后，会显示是否回答正确，并绘制提示信息告诉玩家当前食物的品质变化情况。所有阶段的问题回答完毕后，才会进入食物制作的结算阶段。
