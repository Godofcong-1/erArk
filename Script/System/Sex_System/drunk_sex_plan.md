# 简述
创建一个醉酒系统，具体要点如下：
0. 在 'Script\System\Sex_System' 中创建一个新的文件 'drunk_sex_common.py'，用于存放醉酒系统的公共函数。
1. 醉酒系统的核心是角色的醉酒值，见 'Script\Core\game_type.py' 的 'class Character' 的 'drunk_point'。该数值为int，数值为0~100。
2. 角色的醉酒值有不同的醉酒程度，见 'data\csv\Drunk_Level.csv' 。需要将该csv文件的内容导入到游戏中，作为醉酒程度的参考。导入的代码为 'Script\Config\game_config.py' ，参考该代码中的格式，对该csv进行导入。同样，需要导入食物的醉酒等级csv数据， 'data\csv\Alcohol_Level.csv' 。
3. 醉酒程度的使用方式可以参考角色的睡觉系统中的睡眠程度，参考在 'class Character' 的 'sleep_point' 中的实现方式。需要在醉酒系统的代码中实现一个函数来检测并返回角色当前的醉酒程度，返回的包括醉酒程度的cid和醉酒程度的名称str。
4. 需要在公共函数代码中创建一个函数来查询某个食物会增加的醉酒值，输入食物的id，输出会增加的醉酒值。输入的食物id后，先查找该食物的数据（data\csv\Recipes.csv），在道具数据里查找该食物的醉酒等级，再根据醉酒等级的id查找食物的醉酒等级csv中对应的醉酒值，最后返回该醉酒值的变化值。
5. 需要在公共函数代码中创建一个函数来增加角色的醉酒值，输入角色对象和食物id，先调用上一步的函数获取该食物增加的醉酒值，然后将该醉酒值增加到角色的醉酒值中，并确保醉酒值不会超过100。
6. 需要在公共函数代码中创建一个函数来减少角色的醉酒值，输入角色对象和时间，根据时间的流逝减少角色的醉酒值，确保醉酒值不会低于0。输入的时间单位为分钟，每10分钟基础减少1点醉酒值。时间小于5分钟时不减少醉酒值，时间大于等于5分钟但小于10分钟时按10分钟计算。在基础醉酒值的减少值上，增加根据当前醉酒程度的修正系数，当目前的状态是'醉酒'（根据3中的函数获取）时，系数乘以0.8，当目前的状态是'烂醉'时，系数乘以0.5，其他醉酒状态对系数没有影响。
# 与食物系统的集成
需要与食物系统进行集成，具体要点如下：
1. 在角色行为中，仿照当前的拒绝进食（REFUSE_EAT），新增一个拒绝饮酒（REFUSE_DRINK）的行为，需要有全套的行为数据。
2. 需要在角色吃食物的判断中，即 'Script\System\Cooking_System\food_bag_panel.py' 中的 'eat_food' 增加一个单独的酒类判断，判断的逻辑位置在特殊调味判断之前。该判断函数同样在醉酒的公共函数代码中，函数内容如下：
    - 输入角色对象和食物id，先查找该食物的数据（data\csv\Recipes.csv），在道具数据里查找该食物的醉酒等级，然后根据醉酒等级的id调用 'Script\Design\instuct_judege.py' 中的 'calculation_instuct_judege' 函数来计算实行值，等级的调用名见 'data\csv\InstructJudge.csv'。
    - 该函数根据实行值的判断，返回 True 或 False，True表示可以通过判定，进行后续判定，False表示拒绝饮酒。
    - False时，则将角色当前的行为设置为前面新增的拒绝饮酒（REFUSE_DRINK），将当前食物的属性赋值到行为数据中。
